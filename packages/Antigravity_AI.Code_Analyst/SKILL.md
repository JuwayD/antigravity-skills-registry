# 代码分析与文档生成技能说明 (Code Analyst)

本技能旨在帮助用户深入理解源代码。通过阅读代码，它将生成一份结构化的 Markdown 文档，详细说明代码的作用、核心模块以及其中嵌套的逻辑链。

## 技能核心职责

1. **代码理解**：全篇阅读源代码，识别主要编程语言、框架和库。
2. **作用总结**：简明扼要地概括整个代码库或单个文件的核心目标。
3. **模块划分**：识别代码中的关键组件（如类、函数、服务、数据模型等），并解释其职责。
4. **逻辑链梳理**：追踪数据流和控制流，解释关键业务逻辑是如何从输入运行到输出的。
5. **依赖分析**：理清内部模块间的调用关系及外部依赖。

## 辅助说明文档模板 (推荐结构)

生成文档时应遵循以下结构：

### 1. 概述 (Overview)

- **代码作用**：一句话描述。
- **背景/环境**：所用技术栈 (如 Python, React, Node.js 等)。

### 2. 核心模块 (Core Modules)

使用表格或列表形式列出：

- **模块名称**：文件名或类名/函数名。
- **主要功能**：该模块负责的具体任务。
- **关键接口**：暴露的主要方法或 API。

### 3. 逻辑链分析 (Logic Chain Analysis)

详细描述核心流程。可以使用步骤列表或 Mermaid 流程图。

- **流程启动**：入口点在哪里？
- **数据处理**：中间经历了哪些转换？
- **结果输出**：最终产生了什么？

### 4. 关键算法/技术点 (Key Technical Details)

- 如果有复杂的算法或特殊的设计模式，在此进行重点说明。

## 示例指令

> "请使用 Code Analyst 技能分析 `/path/to/source_code`。我需要一份详细的 Markdown 文档，着重解释数据是如何在各服务之间传递的逻辑链。"

---
**注意**：在分析大型代码库时，请确保先浏览目录结构，再深入分析核心逻辑文件。
